InPrivate Launcher – Privacy Policy

Last updated: May 2026

TL;DR: InPrivate Launcher operates entirely on your device. It does not send any data to external servers. Administrators configure policies locally via Group Policy or registry, and the extension reads them—nothing is transmitted.

What the Extension Does

InPrivate Launcher is a browser extension for Microsoft Edge and Google Chrome that automatically opens specified URLs in InPrivate (Edge) or Guest (Chrome) browsing mode. Organizations can configure URL patterns through Group Policy (GPO) or registry-based policy, and individual users can also set local rules. All configuration stays on your device.

Data Stored Locally

The extension stores the following on your device only:

  • User-configured rules: URL patterns, matching type (exact/prefix/wildcard), and preferences stored in browser local storage.
  • Administrator policy: When deployed by IT, URL rules are read from your system's Group Policy registry—not stored in the extension, read at runtime.
  • Preferences: Theme mode, auto-routing toggle, and maximum rule count.
  • Current tab URL: Read only to match against configured rules; never stored or transmitted.

What We Do Not Do

  • We do not transmit any data over the network.
  • We do not use analytics, telemetry, crash reporting, or any third-party tracking.
  • We do not require user sign-in or accounts.
  • We do not use remote code or external CDNs—all logic is bundled locally.
  • We do not read or interact with websites except to match URLs against configured rules.

Permissions Explained

Permission Purpose
storage Store user-configured URL rules and preferences locally in the browser.
tabs Access the current tab's URL to determine if it matches configured rules and open InPrivate/Guest windows.
windows Open new InPrivate (Edge) or Guest (Chrome) windows when a matching URL is detected.
webNavigation Monitor navigation events to apply rules at the right moment.
<all_urls> Necessary to match URL patterns against any web destination.

Administrator / Enterprise Deployment

When deployed by IT administrators:

  • Administrators configure URL rules through Group Policy (Windows) or manual registry entries.
  • Configuration is stored in your system registry under HKLM\SOFTWARE\Policies\ paths—never transmitted.
  • Users cannot modify administrator-set policies while they are active.
  • When policy is removed, the extension falls back to user-configured local rules.
  • No external policy server or cloud synchronization is used.

Your Data & Control

  • Edit/Delete rules: Manage local rules anytime in the extension's options page.
  • Clear history: No browsing history is stored, but user rules can be deleted at any time.
  • Uninstall: Removes all local storage and preferences associated with the extension.
  • Policy fallback: When administrator policy is removed, your local rules automatically become active again.

Security & Performance

  • The extension uses only standard browser APIs (chrome.storage, chrome.tabs, chrome.windows).
  • No external libraries or CDNs are loaded; all code is bundled at build time.
  • Policy data is read directly from your system registry—not downloaded or cached externally.

Policy Changes

We may update this privacy policy in the future. Any significant changes will be reflected in future extension versions with an updated date at the top of this page.
